# 決定 0001: 回帰ゲートを「ベースライン差分（git stash 比較）」中核に改訂

日付: 2026-07-12
ステータス: **決定（人間 (C) 選択）／ CLAUDE.md・エージェント定義へ反映済み（2026-07-12）**。反映先: `CLAUDE.md`「ゲート（最重要）」節の回帰ゲート行＋「バグ修正ループ」の回帰ゲート行、`.claude/agents/logic-evaluator.md` の検証項目「回帰（ベースライン差分）」＋報告フォーマットのゲート判定行。
文脈: v0.2.3 BUG-1/2・BUG-4・自動更新(b) の回帰確認で、phase3 の3件が「本変更と無関係に」失敗（出荷済み設計とテスト期待の乖離＝`docs/bugs/spec-vs-shipped-booking-flow.md`）。「既存 phase1〜3 全 Green」を絶対条件にすると、本変更起因でない既存赤でクローズできない不整合が露呈した。

## 決定
回帰ゲートの中核判定を、絶対的な「全 Green」から **「ベースライン差分＝変更起因の新規失敗ゼロ」** に改める。
- 手続き（正式化）: ①変更を `git stash` で退避 → ②同スイート再実行 → ③変更あり/なしの失敗集合を比較 → ④差分＝**変更で新たに増えた失敗のみ**を回帰とみなす → ⑤`git stash pop` 後に作業ツリー原状復帰を md5 等で検証。
- 既存条件は維持: **再現テスト Green ＋ Plugin Check 0 errors/0 warnings ＋ 店舗×担当者スコープ隔離ゲート Green**。
- ベースラインにも存在する既存失敗は「別件」として `docs/bugs/` に起票し、回帰ゲートはブロックしない。

## 反映先（ユーザー確認待ち＝設定/CLAUDE.md のため）
planner の運用制約上、`CLAUDE.md` と `.claude/` 配下（エージェント定義）は**エージェント/コーディネータ指示だけでは変更しない**。下記の正確な差分をユーザー確認のうえ適用する。

### (1) CLAUDE.md 「ゲート（最重要）」節 — 回帰ゲート行を差し替え
現行:
> - **回帰ゲート**：再現テスト Green ＋ 既存スイート Green ＋ Plugin Check 0 errors/0 warnings。ここが Green になるまで「修正完了」と扱わない。

改訂案:
> - **回帰ゲート**：再現テスト Green ＋ **ベースライン差分で新規失敗ゼロ**（＝変更を `git stash` 退避した素のコードと同スイートを比較し、変更で新たに増えた赤が無いこと。既存の赤はベースラインにもあれば別件として `docs/bugs/` に起票しブロックしない）＋ Plugin Check 0 errors/0 warnings。ここが Green になるまで「修正完了」と扱わない。手続き詳細は `.claude/agents/logic-evaluator.md`。

### (2) .claude/agents/logic-evaluator.md — 「検証項目」の回帰行を差し替え＋手続き追記
現行 L20:
> - **回帰**：既存 phase1〜3 スイート全 Green（デグレ検出）。

改訂案:
> - **回帰（ベースライン差分）**：既存 phase1〜3 スイートを実行し、**変更起因の新規失敗ゼロ**を判定する。手続き: ①対象の変更を `git stash push -- <変更ファイル>` で退避 → ②同スイート再実行 → ③変更あり/なしの失敗集合を比較 → ④差分（新たに増えた赤）のみを回帰とみなす → ⑤`git stash pop` 後に md5 等で作業ツリー原状復帰を検証（変更が失われていないこと）。ベースラインにも在る既存赤は別件として `docs/bugs/` に起票し、回帰ゲートをブロックしない。

（報告フォーマットの「回帰ゲート: Green / Red」には、判定根拠として「ベースライン比較（新規失敗 N 件 / 既存失敗 M 件）」を併記する運用とする。）
