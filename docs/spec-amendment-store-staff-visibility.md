# Smart Booking 仕様修正: 店舗・担当者のフロント表示制御

## 変更の背景

ターゲット業種の中には、担当者の概念がない（結婚相談所など）や、担当者はいるがユーザーには選ばせたくない（オンライン相談のシフト制など）ケースがある。現在の設計では店舗・担当者が必ず1つ以上存在し、フロントでの表示/非表示は数（1つならスキップ）のみで制御されている。これを管理者が明示的に制御できるように変更する。

## 業種別ユースケース

| 業種 | 店舗表示 | 担当者表示 | 裏の紐づけ |
|------|---------|-----------|-----------|
| 弁護士事務所（1拠点・複数弁護士） | OFF | ON | 弁護士ごとにスケジュール |
| 結婚相談所（1拠点・担当者なし） | OFF | OFF | デフォルト担当者で運用 |
| 整体院（複数院・施術者指名あり） | ON | ON | 院×施術者でスケジュール |
| 学習塾（複数校舎・シフト制） | ON | OFF | 裏で講師にスケジュール紐づけ |

---

## 仕様変更箇所

### 変更1: 設定画面「基本設定」タブに2つのトグルを追加

| 設定キー | ラベル | デフォルト | 説明 |
|---------|--------|-----------|------|
| `smb_show_store_front` | フロントで店舗選択を表示する | ON | OFFにすると、フロント予約フォームで店舗選択ステップが表示されない |
| `smb_show_staff_front` | フロントで担当者選択を表示する | ON | OFFにすると、フロント予約フォームで担当者選択ステップが表示されない |

注意書き（設定画面に表示）:
- 「OFFにしても、管理画面ではスケジュールに店舗・担当者を紐づけて管理できます」
- 「店舗が1つだけの場合、この設定に関わらず店舗選択は自動的にスキップされます」

### 変更2: スキップルール（仕様書 3.1）の改訂

**旧ルール:**

| 条件 | 動作 |
|------|------|
| 店舗が1つのみ | 店舗選択ステップをスキップ |
| 担当者が1人のみ | 担当者選択ステップをスキップ |
| 店舗1つ＋担当者1人 | いきなり日付選択から開始 |

**新ルール:**

| 条件 | 動作 |
|------|------|
| 店舗が1つのみ **OR** `smb_show_store_front` = OFF | 店舗選択ステップをスキップ |
| 担当者が1人のみ **OR** `smb_show_staff_front` = OFF | 担当者選択ステップをスキップ |
| 上記両方に該当 | いきなり日付選択から開始 |

### 変更3: フロントのスキップ時の自動割り当てロジック

店舗選択がスキップされた場合:
- アクティブな店舗が1つ → その店舗を自動選択
- アクティブな店舗が複数 + `smb_show_store_front` = OFF → ショートコードの `store_id` パラメータで指定された店舗を使用。指定なしの場合は `sort_order` 最小の店舗を自動選択
- アクティブな店舗が0 → エラーメッセージ「予約可能な店舗がありません」

担当者選択がスキップされた場合:
- 選択された店舗のアクティブな担当者が1人 → その担当者を自動選択
- 選択された店舗のアクティブな担当者が複数 + `smb_show_staff_front` = OFF → 日付・時間で空いている担当者を自動割り当て（最初に見つかった空き枠の担当者）
- 選択された店舗のアクティブな担当者が0 → エラーメッセージ「予約可能な担当者がいません」

### 変更4: 担当者非表示時のスケジュール取得ロジック

`smb_show_staff_front` = OFF かつ担当者が複数いる場合:
- フロントのカレンダーには、選択された店舗の **全担当者のスケジュールを統合** して空き枠を表示する
- 同じ日時に複数の担当者が空いていれば、その時間枠の capacity は合算される
- 予約確定時に、空いている担当者の中から1人を自動割り当てする（sort_order 順で最初に空きのある担当者）

### 変更5: DB設計は変更なし

- smb_stores, smb_staff のスキーマは変更しない
- store_id, staff_id は引き続き NOT NULL
- プラグイン有効化時にデフォルトの店舗1つ・担当者1つを自動作成する動作は変更しない
- 新しい設定値は wp_options に保存（既存の smb_ プレフィックスのオプションと同様）

### 変更6: 確認画面・完了画面・メール通知の表示

- `smb_show_store_front` = OFF の場合: 確認画面・完了画面・メール本文に店舗名を表示しない
- `smb_show_staff_front` = OFF の場合: 確認画面・完了画面・メール本文に担当者名を表示しない
- メールテンプレート変数 `{store_name}` `{staff_name}` は、非表示設定でも空文字ではなく値を持つ（管理者宛メールでは内部情報として必要）
- ユーザー宛メールのデフォルトテンプレートでは、非表示設定時にその行を出力しない

### 変更7: 管理画面への影響

管理画面の以下の機能は変更なし（フロント表示制御はフロントのみに影響）:
- 店舗・担当者のCRUD
- スケジュール管理（店舗・担当者フィルタ）
- 予約一覧（店舗・担当者カラム）
- CSV出力（店舗・担当者カラム）
- 手動予約作成

### 変更8: ショートコード `store_id` パラメータの挙動

- `[smart_booking store_id="2"]`: 従来通り、特定店舗の予約フォームのみ表示
- `smb_show_store_front` = OFF + `store_id` 指定あり: その店舗を自動選択（店舗選択画面は表示しない）
- `smb_show_store_front` = OFF + `store_id` 指定なし: sort_order 最小の店舗を自動選択
