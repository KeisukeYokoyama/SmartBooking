# 仕様追補 v0.4.2 — カスタムフィールドのメール変数展開

対象バージョン: v0.4.2（外部ユーザー報告1への対応・案A）
起票: 2026-07-16
位置づけ: `docs/smart-booking-spec.md` の §5.5（テンプレート変数）／§8.3（未定義変数の扱い）に対する**追補**。
歴史記録の据え置きポリシーにより spec 本体は編集せず、差分を本メモに記録する。
背景・調査の正本は `docs/bugs/v0.4.2-external-report-ledger.md`。

## 1. 変更の要旨

これまでメール本文・件名で展開できるテンプレート変数は固定8変数のみだった：

```
{customer_name} {customer_email} {customer_phone} {reservation_id}
{schedule_date} {schedule_time} {store_name} {staff_name}
```

v0.4.2 から、**予約フォームのカスタムフィールドの回答値も `{field_key}` 変数として展開**できる。
管理者が「フォーム設定」で追加した任意項目（例: `field_4` = 資料送付）の回答を、
受付メール／承認メールに差し込めるようになった。

実装: `Smart_Booking_Reservation_Context::template_vars()`／`custom_field_vars()`。
公開契約（REST 形・DB スキーマ）は無改修。**マイグレーション不要**（保存済み meta を遡って参照するだけ）。

## 2. §5.5 追補: 展開されるカスタム変数

予約に紐づく `form_id` のカスタムフィールド定義（`smart_booking_custom_fields`）を走査し、
`smart_booking_reservation_meta` の回答値から次の変数を生成する。

| フィールド種別 | 生成される変数 | 値 |
|---|---|---|
| text / textarea / email / tel / select / radio | `{field_key}` | 回答文字列そのまま |
| checkbox（複数選択） | `{field_key}` | 選択値を「、」で結合（予約詳細の表示と同じ結合記号） |
| address（郵便番号＋住所） | `{field_key}_zip` | 正規化7桁の郵便番号（ハイフン無し） |
| 〃 | `{field_key}_address` | 住所文字列 |
| 〃 | `{field_key}` | `〒{郵便番号} {住所}`（確認画面・予約詳細と同じ結合形式・空要素は除外） |

- **スコープ**: 展開対象はその予約が使ったフォーム（`reservations.form_id`）の定義に限定する。
  複数フォーム（v0.4.0）では同じ `field_key` が別フォームに存在し得るため、他フォームの定義は混ぜない。
- メールテンプレート（`smart_booking_mail_*`）はフォーム横断のグローバル設定のため、
  同じ `{field_key}` でも**予約に使われたフォームの回答**で置き換わる。管理画面の変数ヘルパーは
  この点を明示し、変数をフォーム別グループで提示する。

## 3. §8.3 追補: 未定義・非表示・衝突の扱い

- **表示条件（機能③）で非表示だったフィールド**: meta 行が無いため**空文字**に展開する
  （未入力と同じ扱い。§8.3「未定義変数は空文字」の原則に沿う）。
- **固定8変数との衝突**: `field_key` が固定変数名（`store_name` 等）と一致した場合は
  **固定8変数を優先**する（`template_vars()` はカスタム変数を先に組み、固定変数で上書きする）。
- **予約語の防御**: 新規フィールド作成時、`field_key` に固定8変数名を指定すると
  `smb_field_key_reserved`（400）で拒否する（`Smart_Booking_REST_Custom_Fields::RESERVED_TEMPLATE_KEYS`）。
  空欄指定時は従来どおり `field_N` を自動採番する（予約語には該当しない）。

## 4. 関連 UX 改善（同梱）

- カスタムフィールドの `field_key` を**任意入力**化。日本語ラベルのみでもキー欄を空にして作成でき、
  サーバが `field_N` を採番する（`CustomFieldModal`／`sanitize_create`）。
- 「フォーム設定」のフィールド一覧に「メール変数 `{key}` として使用可」のヒントを表示。
- 「設定 > メール通知」の変数ヘルパーに、フォーム別のカスタム変数チップを追加（クリックで本文に挿入）。
- `CustomFieldModal` のキー欄の説明文を、実装（メール変数として使える）に一致する文言へ修正。

## 5. デグレ非発生の担保

- カスタムフィールドを使っていない既存テンプレートの出力は**従来と1文字も変わらない**
  （固定8変数の展開順・値は不変。カスタム変数は追加されるだけで固定変数を上書きしない）。
- `render()`（`strtr` による一括置換）のロジックは不変。longest-key-first により
  `{key}_zip` と `{key}` は正しく区別される。
